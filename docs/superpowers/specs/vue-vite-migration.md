# Vue Vite Migration Spec

## Goal

Migrate Tycho's browser UI from plain HTML/CSS/JavaScript to a Vue 3 + Vite frontend while preserving the existing product behavior and security boundaries.

## Requirements

- The browser app is authored as Vue components and built by Vite.
- The existing project management workflow remains available:
  - select a configured project
  - add a managed project with name, local path, and description
  - see validation errors for invalid paths
  - delete managed projects
- The existing session workflow remains available:
  - create allowlisted agent sessions for the selected project
  - display session list entries
  - mount xterm terminals for tuimux panes
  - send input, resize, focus, and close events through the existing WebSocket contract
- The browser continues to call only Tycho's own HTTP/WebSocket APIs.
- The browser still cannot send arbitrary shell commands, arbitrary environment variables, or arbitrary agent commands.
- Production build output is generated by Vite and served by Express.
- Development mode serves the Vue app through Vite middleware while retaining the existing Express/WebSocket server.

## Non-Goals

- No visual redesign.
- No router or multi-page application.
- No change to tuimux protocol semantics.
- No direct browser integration with model providers.

## Test Coverage

- E2E must prove the Vue/Vite client boots.
- E2E must keep covering project add/delete happy path.
- E2E must keep covering invalid project path failure path.
- `scripts/verify` must cover server typecheck, Vue typecheck, unit tests, E2E, and production build.
